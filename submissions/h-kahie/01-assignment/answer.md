# Introduction to Data Science and Machine Learning

---

## 1. Data Science and Machine Learning + Relationship

Data Science is an interdisciplinary field that focuses on extracting knowledge and insights from structured and unstructured data using scientific methods, statistics, and computing techniques. It combines domain knowledge, programming, and analytical thinking to turn raw data into useful information for decision-making.

Machine Learning (ML) is a subfield of Artificial Intelligence (AI) that enables systems to learn patterns from data and make predictions or decisions without being explicitly programmed.

Machine Learning is a core component of Data Science. Data Science focuses on the entire process of working with data (collection, cleaning, analysis, and interpretation), while Machine Learning is mainly used in the modeling stage to build predictive systems.

**Example:**
In Somalia, Data Science can be used to analyze historical weather data, rainfall patterns, soil conditions, and drought records. Machine Learning models can then predict the likelihood of droughts or abnormal weather changes, helping government and aid organizations prepare early warnings and reduce the impact of recurring climate disasters.

---

## 2. Data Science Lifecycle and Role of Machine Learning

The Data Science lifecycle includes:

1. Problem Definition  
2. Data Collection  
3. Data Cleaning and Preprocessing  
4. Exploratory Data Analysis (EDA)  
5. Model Building  
6. Model Evaluation  
7. Deployment  
8. Monitoring and Maintenance  

Machine Learning is mainly applied in the **Model Building stage**, where algorithms learn patterns from data and generate predictions.

---

## 3. Supervised vs Unsupervised Learning

### Supervised Learning
- Uses labeled data  
- Used for prediction tasks  
- Example: Spam detection, house price prediction  

### Unsupervised Learning
- Uses unlabeled data  
- Used for finding hidden patterns  
- Example: Customer segmentation, clustering  

---

## 4. Overfitting

Overfitting happens when a machine learning model memorizes training data instead of learning general patterns.

### Causes:
- Too complex models  
- Small datasets  
- Too many features  

### Prevention:
- Use more data  
- Simpler models  
- Regularization (L1, L2)  
- Cross-validation  

---

## 5. Training and Test Data Split

Datasets are divided into:

- Training set (80%) → used to train the model  
- Test set (20%) → used to evaluate the model  

### Why?
To ensure the model works well on unseen data and does not just memorize training data.

**Analogy:**
- Training = practice exam  
- Test = final exam

## 6. Case Study: Machine Learning in Healthcare

Machine Learning is used in healthcare to predict diseases like diabetes and heart disease.

### How it works:
- Patient data is collected  
- ML model learns patterns  
- Predicts disease risk  

### Impact:
Helps doctors detect diseases early and improve treatment decisions.

### Lifecycle stages involved:
- Data Collection  
- Data Cleaning  
- Model Building  
- Evaluation  
- Deployment
